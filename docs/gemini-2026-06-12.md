# Security Audit Report - 2026-06-12

## Summary

I have conducted a security review of the project, focusing on common web vulnerabilities and the specific implementation details of this application. Overall, the project demonstrates a strong security posture with robust protections for authentication, data handling, and privacy.

## Findings

### 1. Privacy: Gravatar Integration (Verified Intentional)

- **Status**: Not a Vulnerability / User Preference
- **Location**: `app/models/user.rb`, `app/helpers/users_helper.rb`
- **Description**: The application uses Gravatar to display user avatars. This involves exposing the MD5 hash of the user's email address in the image URL.
- **Verification**: Investigation confirmed that the application only exposes the real MD5 hash if a Gravatar image actually exists for that email (verified via `gravatar_exists?`). If no Gravatar exists, a bogus hash is used, preventing the leak of hashes for non-Gravatar users.
- **Justification**: Users who have set up a Gravatar for their email address generally expect and consent to its use across services where they register with that email. Since the application respects this by only enabling it when an image is present, this is considered an intentional feature rather than a vulnerability.

### 2. Authorization: Editor Management (Verified Intentional)

- **Status**: Not a Vulnerability
- **Location**: `app/controllers/projects_controller.rb#update_additional_rights`
- **Description**: The application allows any user with "edit" permissions to add other editors to the project. While only "controllers" (owners/admins) can remove editors, the ability for editors to grant access to others was reviewed.
- **Justification**: This is confirmed as expected and desired behavior to facilitate collaboration. Once a user is trusted as an editor, they are also trusted to expand the collaboration team as needed.

### 3. Rate Limiting: Hardening against Misconfiguration and Spoofing

- **Status**: Targeted for Hardening
- **Location**: `app/lib/client_ip.rb`, `config/initializers/rack_attack.rb`, `config/application.rb`
- **Description**: The current `ClientIp.acquire` method uses a hardcoded index (`[-2]`) to extract the user's IP from the `X-Forwarded-For` header. This is brittle and susceptible to spoofing.

#### Decision: Option 2 (Rails Standard `trusted_proxies` with Dynamic Loading)

We have selected **Option 2** with an added layer of resilience: **Dynamic Loading with Static Fallback**. This ensures the system stays up-to-date with Fastly's infrastructure changes while remaining reliable even during service interruptions (e.g., if the Fastly API is unavailable).

#### Implementation Plan:

**1. Environment Variables**
- `TRUSTED_PROXIES_URL`: Set to `https://api.fastly.com/public-ip-list`.
- `TRUSTED_PROXIES`: Static fallback list of Fastly CIDRs.
  - *Current Values*: `23.235.32.0/20,43.249.72.0/22,103.244.50.0/24,103.245.222.0/23,103.245.224.0/24,104.156.80.0/20,140.248.64.0/18,140.248.128.0/17,146.75.0.0/17,151.101.0.0/16,157.52.64.0/18,167.82.0.0/17,167.82.128.0/20,167.82.160.0/20,167.82.224.0/20,172.111.64.0/18,185.31.16.0/22,199.27.72.0/21,199.232.0.0/16,2a04:4e40::/32,2a04:4e42::/32`
- `TRUSTED_PROXIES_DISABLED`: If `true`, bypasses all checks (useful for private networks).
- `ENFORCE_ORIGIN_SHIELDING`: If `true`, rejects requests that bypass Fastly.

**2. Detailed Method Design (`SecurityUtils`)**
Logic is split into small, testable methods in `lib/security_utils.rb`. This design leverages `security_assertion`, a custom "fail-fast" method already defined in `lib/security_utils.rb` that raises a `SecurityAssertionError` on failure. Note the use of explicit parameters to minimize environment-dependency during testing.

```ruby
# Orchestrates loading.
def self.load_trusted_proxies(url: nil, static: nil, fail_fast: false, disabled: false)
  return [].freeze if disabled

  ips_dynamic = fetch_dynamic_proxies(url)
  ips_static = parse_static_proxies(static)

  warn_if_mismatched(ips_dynamic, ips_static, url)

  ips = ips_dynamic.empty? ? ips_static : ips_dynamic
  ensure_proxies_present(ips, fail_fast)

  ips.map { |ip| IPAddr.new(ip) }.freeze
end

# Attempts fetch from URL.
def self.fetch_dynamic_proxies(url)
  return [] if url.blank?
  begin
    response = HTTParty.get(url, timeout: 5)
    return [] unless response.success?
    json = response.parsed_response
    (json['addresses'] || []) + (json['ipv6_addresses'] || [])
  rescue StandardError => e
    Rails.logger.warn "Dynamic proxy load failed: #{e.message}. Falling back to static list."
    []
  end
end

# Parses static string.
def self.parse_static_proxies(static)
  return [] if static.blank?
  static.split(',').map(&:strip)
end

# Logs a warning if dynamic and static data differ.
def self.warn_if_mismatched(dynamic, static, url)
  return unless dynamic.present? && static.present?
  return if dynamic.sort == static.sort

  Rails.logger.warn "SECURITY WARNING: TRUSTED_PROXIES (static) is OUT OF DATE. " \
                    "Dynamic data from #{url} differs from the static fallback."
end

# Ensures at least one proxy is present if fail_fast is enabled.
def self.ensure_proxies_present(ips, fail_fast)
  return unless ips.empty? && fail_fast
  security_assertion(false, "No trusted proxies configured via URL or static list.")
end

# Checks if request passed through at least one edge proxy.
def self.origin_shielded?(request, edge_proxies, disabled: false, enforced: false)
  return true if disabled || !enforced
  # ActionDispatch::RemoteIp stores the proxy list in the environment
  chain = request.get_header('action_dispatch.remote_ip').send(:proxies)
  chain.any? { |ip| edge_proxies.any? { |proxy| proxy.include?(ip) } }
end
```

**3. Documentation & Inline Comments**
The following comment must be added to `lib/security_utils.rb`, `config/application.rb`, and `app/lib/client_ip.rb`:
> "The trusted proxies' IP addresses are *not* being used for user authentication; they're being used to counter CDN piercing and to ensure that our rate limits apply to the correct IP addresses."

**4. Boot-time Configuration (`config/initializers/fastly.rb`)**
```ruby
# Configure Rails trusted proxies
Rails.application.config.action_dispatch.trusted_proxies = SecurityUtils.load_trusted_proxies(
  url: ENV['TRUSTED_PROXIES_URL'],
  static: ENV['TRUSTED_PROXIES'],
  fail_fast: Rails.env.production?,
  disabled: ENV['TRUSTED_PROXIES_DISABLED'] == 'true'
)
```

**5. Startup Middleware Order Verification (`config/initializers/rate_limiting_check.rb`)**
```ruby
remote_ip_idx = Rails.application.config.middleware.index(ActionDispatch::RemoteIp)
rack_attack_idx = Rails.application.config.middleware.index(Rack::Attack)
if remote_ip_idx.nil? || rack_attack_idx.nil? || remote_ip_idx > rack_attack_idx
  raise "Security Configuration Error: RemoteIp must precede Rack::Attack in middleware stack."
end
```

**6. Simplify `ClientIp` (`app/lib/client_ip.rb`)**
```ruby
def self.acquire(req)
  req.remote_ip
end
```

**7. Anti-Piercing Enforcement (`ApplicationController`)**
```ruby
before_action :verify_origin_shielding

def verify_origin_shielding
  edge = Rails.application.config.action_dispatch.trusted_proxies
  return if SecurityUtils.origin_shielded?(
    request, edge,
    disabled: ENV['TRUSTED_PROXIES_DISABLED'] == 'true',
    enforced: ENV['ENFORCE_ORIGIN_SHIELDING'] == 'true'
  )
  render plain: "403 Forbidden - Direct origin access not allowed", status: :forbidden
end
```

#### Testing Strategy (100% Statement Coverage):

We will use `test/unit/lib/security_utils_proxy_test.rb` to verify logic. By passing explicit parameters to the methods, we avoid the need for complex environment stubbing:
- **Unit Tests**:
  - Test `load_trusted_proxies` with all combinations of `url`, `static`, `fail_fast`, and `disabled`.
  - Test `parse_static_proxies` with various comma-separated strings (spaces, malformed, empty).
  - Test `origin_shielded?` by passing mock requests and proxy lists.
- **Integration Tests**:
  - Verify the application boots (or fails) correctly under simulated production environments.
  - Verify the middleware order check using `Rails.application.config.middleware`.

### 4. General Observations (Verified Safe)

- **SQL Injection**: Search logic in `ProjectsController` and `Project` model scopes is well-protected using Arel's parameterization and `sanitize_sql_like`.
- **XSS**: High-risk Markdown areas were previously audited as safe. The strict CSP policy (no `unsafe-inline`) provides a strong second layer of defense.
- **Open Redirects**: The application uses robust guards (`valid_return_path?`, `allow_other_host: false`) for redirections.
- **Mass Assignment**: Sensitive fields like `user_id` and `role` are protected by explicit checks and filtered `project_params`.

## Conclusion

The application follows modern security best practices. The identified items are primarily minor privacy or logic refinements rather than critical vulnerabilities.
